Submission Requirements

Complete these requirements to properly submit your project for judging. Make sure to follow all guidelines to avoid disqualification.

Submission Checklist

Before submitting your project, please ensure that you have completed all the necessary requirements listed below.

  • GitHub Repository

    Ensure your project GitHub repository is set to public so the judges can review your code.

  • Demo Video (5 Minutes Max)

    Provide a link to your recorded demo video showcasing your project, key features, and problem-solving approach. Keep it under 5 minutes.

  • README.md File

    Your repository must contain a well-documented README.md that includes:

    • Project title and description
    • Technologies used
    • Setup and installation instructions
    • Team members and their contributions
    • Link to the demo video

  • Submission via Devpost

    Ensure you submit your GitHub repository link and demo video through Devpost before the deadline.

  • Deadline Compliance

    Late submissions will not be considered. Verify that all materials are uploaded before the deadline.

Important

Double-check all links and documentation to avoid disqualification. If you encounter issues, reach out via the hackathon support channels.

Final Verification Checklist

  • GitHub repository is public and contains all project files
  • Demo video is under 5 minutes and clearly demonstrates your project
  • README.md is complete with all required information
  • All links are working correctly
  • Submission has been completed on Devpost before the deadline
