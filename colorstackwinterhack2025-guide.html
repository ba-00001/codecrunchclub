How to Submit

Follow these steps carefully to submit your project for judging. Review the hack agenda regularly for updates.

1. Create Your GitHub Repository

Create a public GitHub repository named: colorstackwinterhack2025-[projectname]

Your repository must include:

  • • A complete README.md with project description
  • • Setup & installation instructions
  • • Usage instructions and features
  • • Technologies used
  • • Explanation of how your project solves the problem you’re targeting

Upload a demo video (MP4, max 5 minutes) to the repository and include screenshots.

Attach a presentation file (.ppt, .pptx, or .pdf) in the repo. Also include a public Google Slides link for the live demo.

Live Demos are 4 minutes + 1 minute Q&A.
Meeting link: will be posted on Slack.

Presentation Slide Requirements

  • • Slide 1: Project name, team members, and track(s)
  • • Slide 2: Description of the problem you are solving and how your project addresses it

2. Add Your Demo Video

Provide a five-minute demo video showcasing functionality. Include the link in your README and upload the MP4 file directly into your repository.

3. Finalize Your README

Your README must include:

  • • Project title & description
  • • How your project aligns with solving the chosen problem
  • • Technologies used
  • • Setup & installation instructions
  • • Team members & contributions
  • • Demo video link & screenshots

Note: Missing, broken, or low-quality videos may result in reduced scoring.

4. Submit on Devpost

Submit your GitHub repository link and complete your Devpost submission any time before the final hard submission deadline. You are encouraged to make your first submission early and continue updating it as you improve your project, up until submissions close.

Judging Criteria

Projects will be evaluated across six categories, each scored 1–5. The final score is a weighted average of all categories.

Technical Difficulty

Complexity of the technical challenges solved and sophistication of implementation.

Functionality

How well the solution works, reliability, completeness, and bug-free performance.

Innovation & Creativity

Originality, creativity, and uniqueness of the approach.

Design & User Experience

Usability, accessibility, visual appeal, and overall user-centered design.

Impact & Usefulness

Practical value, real-world relevance, and meaningfulness of the solution.

Presentation & Demo

Clarity, delivery, confidence, and quality of live demonstration.

Criteria Weighting

Judging Process

  1. 1. Initial Screening: Checks rules, deadlines, track selection, repo access.
  2. 2. Project Review: Judges evaluate code, README, video, slides.
  3. 3. Individual Scoring: Each judge rates all six categories.
  4. 4. Deliberation: Judges compare results and determine winners.
  5. 5. Announcement: Winners announced on slack channel & Devpost.
